# 📚 온라인 교육 플랫폼 이탈률과 수료율 분석

**국비교육 실전 프로젝트** | 2026.03.18 ~ 2026.03.31 | 

---

## 📌 프로젝트 개요

> 하버드·MIT가 공동 설립한 세계적인 MOOC 플랫폼 **edX**의 실제 수강 데이터를 활용하여, 단계별 학습자 이탈률과 행동 패턴을 분석하고 수료율 개선 방안을 도출한 프로젝트입니다.
> 

**문제 상황**

- MOOC 플랫폼의 고질적 문제인 낮은 수료율 — 전체 학습자 중 수료자는 단 **2.8%**
- 학습 완료율 중앙값 12.6% 수준으로, 등록자 상당수가 강의를 시작조차 하지 않음
- 플랫폼 학습자 활동 현황을 한눈에 파악할 수 있는 대시보드 부재

**분석 목표**

- 퍼널 분석(등록 → 열람 → 탐색 → 수료)을 통한 단계별 학습자 행동 패턴 파악
- 단계별 이탈 원인과 수료에 영향을 미치는 핵심 행동 지표 도출
- 플랫폼 운영자·콘텐츠 제작자 관점의 Tableau 대시보드 구축

**주요 컬럼 4개 영역**

| 영역 | 컬럼 |
| --- | --- |

**퍼널 구조 (학습자 유형)**

| --- | --- |

| EXPLORER | 절반 이상 시청했지만 수료하지 못한 학습자 |
| COMPLETER | 수료하고 이수증을 취득한 학습자 |

## 🔍 분석 과정

### 1️⃣ 전처리 (Preprocessing)

- **불필요 컬럼 제거**: roles, nplay_video 등
- **컬럼명 정리**: 한글 → 영문 변환, 소문자 통일 (final_cc_cname_DI → country 등)
- **결측치 처리**: grade → fillna(0), gender/loe → 'unknown', yob → 각 그룹 중앙값
- **이상치 처리 3가지 케이스**: 성적 0인데 수료한 경우 / 열람 없이 심화 학습한 경우 / 열람 없이 챕터 기록 있는 경우 제거
- **파생변수 생성**: course_id 분리(institution, course_code, course_year, course_category) / duration(수강 기간) / age(나이) / student_type(no_viewer, viewer, explorer, completer)

### 2️⃣ EDA 및 통계 분석

**플랫폼 현황**

- 전체 수료율: **2.8%** (수료 17,680명 / 미수료 619,562명)
- 20대 학습자 수 압도적 (41만명), 수료율은 10대(2.9%)가 가장 높음
- 남성 학습자 수 가장 많으나(40만명), 수료율은 성별 미상(4.18%) > 여성(2.98%) > 남성(2.41%)
- 컴퓨터 공학 강의 선호도 가장 높지만 수료율은 **1.7%로 가장 낮음**

**퍼널 단계별 이탈 현황**

| 단계 | 잔류율 | 이탈률 |
| --- | --- | --- |
| 등록 → 열람 | 62.7% | 37.3% |
| 열람 → 탐색 | 6.3% | **89.9%** ← 최대 병목 |
| 탐색 → 수료 | 2.8% | 56.2% |

**왜 탐색 단계로 못 가는가?**

- 조회 그룹의 **61%가 1~2일 이내 이탈** → 무료 강의 특성상 학습 지속 동기 부재
- 조회 그룹 활동일수 중앙값 7일 vs 탐색 그룹 120일 → **17배 차이**
- Kaplan-Meier 생존 분석: **전체의 84%가 5일 이내 이탈** → 수료율 개선의 최대 병목은 '초기 이탈'

**수료를 결정하는 요인은?**

- 탐색 챕터 11개 이상 구간에서 수료율 **+44%p 급등**
- 로그변환 T-Test + Mann-Whitney U Test → 수료자와 이탈자를 가르는 결정적 차이는 **접속 빈도(ndays_act)와 누적 활동량(nevents)**
    - nevents 효과크기(Cohen's d): **1.237 (매우 큰 효과)**
    - ndays_act 효과크기: **1.323 (매우 큰 효과)**
- 탐색 vs 수료 그룹 성적 차이: Mann-Whitney U Test p-value = 0, 효과크기 r = **0.993**

### 3️⃣ 대시보드 (Tableau)

두 가지 페르소나 관점으로 대시보드 구성:

**플랫폼 운영자 대시보드**: KPI 현황 / 단계별 이탈 현황 / 국가·성별·학력별 수료율 / 활동일수별 수료율 / 수료율 상위 5개 강의

**콘텐츠 제작자 대시보드** (내 담당): 강의별 수료율·평균 성적 / 단계별 이탈 현황 / 연령대별 학생 현황 / 활동지표(활동횟수·활동일수·탐색챕터수) 시계열 / 레이더 차트로 Student Type별 비교

## 📈 핵심 인사이트

> 💡 **수료율 개선의 핵심은 '완주율' 보다 '초반 활성화와 지속성' 관리**
> 

**제품/운영 관점**

- 등록은 유입 지표, viewed/explored는 활성화 지표, certified는 성과 지표로 **분리해서 관리** 필요
- 같은 이탈이라도 강사는 콘텐츠 구조·난이도로, 운영자는 온보딩·리마인드·탐색 UX로 해결 → **역할 분담** 필요

**지표/측정 관점**

- KPI 분모를 전체 등록자로 설정하면 의사결정 왜곡 → **활동자 기준 KPI** 필요
- 수료율 개선보다 **열람 → 탐색 전환율** 향상이 현실적인 접근

**개선 방안 제안**

| 단계 | 개선 방안 |
| --- | --- |
| 열람 → 탐색 | 강의 등록 후 **3일 미접속 학습자 대상 알림 메시지** 전송 |
| 열람 → 탐색 | **1~2강 내 간단한 퀴즈·미니 실습** 구성으로 조기 성취감 제공 |
| 탐색 → 수료 | 수료 조건에 **시험 성적 + 학습 참여도** 함께 반영하는 방식으로 개선 |

## 💬 튜터 피드백 & 개선사항

### ✅ 잘한 점

- edX 플랫폼 특성을 세 가지 축(하버드·MIT 설립 / 무료 청강 / 마이크로마스터즈 인정)으로 간결하게 정리해 **배경지식 없는 청중도 바로 이해할 수 있는 구성**
- 수료율 중앙값 12.6%라는 기존 연구 수치를 인용해 문제의 심각성을 뒷받침하고, **퍼널 분석 + 행동 지표 + 대시보드 구축**이라는 세 가지 분석 목표로 구체화한 점 우수
- **플랫폼 운영자 / 콘텐츠 제작자** 두 페르소나를 사전에 설정해 분석 활용 대상을 명확히 한 점이 실무적 관점에서 탁월
- student_type을 no_viewer / viewer / explorer / completer로 정의하고 각 유형의 의미를 명확히 제시해 **퍼널 분석의 토대를 잘 마련**
- 퍼널 논리에 맞지 않는 이상치 3가지 케이스를 식별하고 처리한 **데이터 논리적 정합성** 우수
- **열람→탐색(89.9%) 병목 구간 식별** 및 조회 그룹의 61%가 1~2일 이내 이탈한다는 구체적 수치 기반 개선안 도출
- **Kaplan-Meier 생존 분석**으로 전체 84%가 5일 이내 이탈함을 시각적으로 제시한 점 인상적
- 로그변환 T-Test + Mann-Whitney U Test 병행으로 결과 강건성 확인, Cohen's d와 효과크기 r 함께 제시해 **p-value만으로 판단하지 않은 통계적 접근 우수**
- 운영자·콘텐츠 제작자 두 대시보드가 **명확히 차별화**된 구성으로 프로젝트 완성도를 크게 높임
- PPT 디자인이 깔끔하고 코드 정리가 잘 되어 있음

### 🔧 개선사항

**분석 측면**

- **퍼널 단계 명칭 통일 필요**: PPT(Registered/Viewed/Explored/Certified)와 코드(no_viewer/viewer/explorer/completer) 명칭이 혼재 → 발표용·분석용 용어 하나로 통일 필요
- **샘플 수 표시 추가**: 연령대별 분석에서 70대 이상 등 표본이 작은 집단은 비율 해석이 과도해질 수 있어 구성 비율 그래프에 **n(샘플 수) 함께 표시** 필요
- **연령/성별/학력/국가와 student_type 관계에 카이제곱 검정 추가** → 세그먼트 차이를 통계적으로도 검증하면 분석이 더 탄탄해짐
- **3일 미접속 알림 근거 명시**: 생존 곡선상 3일차부터 70% 이상 이탈 발생 → 근거를 발표 자료에 명시하고, 여러 개선안 중 **ROI 가장 높은 액션 1개를 핵심으로 강조** 필요

**전처리 측면**

- **전처리 비포·애프터 문서화**: 각 전처리 규칙이 몇 행에 영향을 줬는지 행 수·컬럼 수 변화를 before-after로 명시 필요
- **결측치 보정 방식 수정 (중요 ⚠️)**: 현재 nevents, ndays_act, nchapters 결측치를 course_id + viewed + **explored + certified 기준 중앙값**으로 채운 것은 결과 변수로 설명 변수를 채운 셈 → **데이터 누수 위험** 존재. explored/certified를 그룹화 기준에서 제외하고 결측치 보정 방식 수정 필요
- **이상치 제거 단위 검토**: 현재 userid 단위로 통삭제 시, 한 사용자가 여러 코스 행을 가질 수 있어 정상 행까지 함께 제거될 위험 → **row 단위 제거 가능 여부 먼저 검토** 후 적용 필요

**코드 측면**

- **마크다운 설명 보강**: 각 셀에 처리 이유, 지표 계산 방식, 어떤 슬라이드와 연결되는지 추적할 수 있는 마크다운 주석 추가 필요
- **nplay_video 제거 사유 명시**: p.11에서 EDA 차트로 분포를 보여준 뒤 p.14에서 불필요 컬럼으로 제거 → 흐름상 혼동이 생기므로 **제거 사유 한 줄이라도 명시** 필요
